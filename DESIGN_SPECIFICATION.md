# Software Design Specification: Plus Game Zone

## 1. Introduction

### 1.1 Purpose
The purpose of this document is to provide a comprehensive software design specification for the **Plus Game Zone** financial management application. It outlines the architecture, data models, functional components, and user interface design to guide development and maintenance.

### 1.2 Scope
Plus Game Zone is a professional financial management and growth hub designed for business partners. It features sales tracking, expense management, Equb (ROSCA) tracking, loan management, asset tracking, and AI-powered financial insights.

### 1.3 Overview
The application is built as a modern Single Page Application (SPA) using React and Vite. It prioritizes a high-density, professional "command center" aesthetic, optimized for both desktop and mobile use.

---

## 2. System Architecture

### 2.1 Technology Stack
- **Frontend**: React 19, TypeScript
- **Build Tool**: Vite
- **Styling**: Tailwind CSS (Utility-first CSS)
- **Icons**: Lucide React
- **Charts**: Recharts
- **AI Integration**: Google Gemini API (@google/genai)
- **Animations**: Motion (framer-motion)

### 2.2 State Management
The application uses a decentralized state management approach:
- **React Hooks**: `useState` and `useEffect` for local and shared state.
- **LocalStorage**: Persistent storage for user data, transactions, and settings, ensuring data survives page refreshes in the absence of a dedicated backend.

### 2.3 Security Model
- **PIN-based Authentication**: Users access the system via an email and a 4-digit PIN.
- **Role-Based Access Control (RBAC)**:
    - `SUPER_ADMIN`: Full access to all management features.
    - `ADMIN`: Operational access (sales, expenses, reports).
    - `MEMBER`: Audit/View-only access to specific modules.
- **Security Overlay**: A lock screen that triggers after inactivity or manual lock.
- **Masking Mode**: Ability to hide sensitive financial figures in the UI for privacy in public spaces.

## 3. Data Models

### 3.1 User
- `id`, `name`, `role`, `pin`, `email`, `equityPercentage`, `totalDividendsPaid`, `joinedDate`.

### 3.2 Transaction
- `id`, `amount`, `type` (INCOME, EXPENSE, TRANSFER, etc.), `method` (CASH, TELEBIRR, CBE, EBIRR), `category`, `note`, `vendor`, `date`, `isAutoGenerated`.

### 3.3 Equb (ROSCA) Group
- `id`, `name`, `membersCount`, `currentRound`, `contributionAmount`, `frequency`, `startDate`, `myTurnIndex`, `status`.

### 3.4 Loan
- `id`, `loanName`, `lenderName`, `totalAmount`, `netDisbursed`, `interestRate`, `durationMonths`, `monthlyRepayment`, `remainingAmount`, `nextPaymentDate`.

### 3.5 Asset
- `id`, `name`, `value`, `category` (REAL_ESTATE, VEHICLE, EQUIPMENT, etc.), `acquisitionDate`.

### 3.6 Goal
- `id`, `title`, `type` (SAVINGS, LOAN_PAYOFF, ASSET_PURCHASE), `targetAmount`, `currentAmount`, `deadline`, `status`.

---

## 4. Functional Requirements

### 4.1 Dashboard
- Real-time summary of total balance, monthly income vs. expenses.
- Visual charts (Area/Bar) for financial trends.
- Quick action status for Equb and Loans.

### 4.2 Transaction Management
- Record sales (Income) and expenses with categories and payment methods.
- Support for balance transfers between different wallets (e.g., Cash to Telebirr).
- Transaction history with filtering and search.

### 4.3 Equb Tracker
- Manage traditional Ethiopian ROSCA (Equb) groups.
- Track rounds, contributions, and payout turns.

### 4.4 Loan & Debt Management
- Track business loans, interest rates, and repayment schedules.
- Visual progress bars for loan payoffs.

### 4.5 Asset Management
- Inventory of business assets with valuation and acquisition tracking.

### 4.6 Growth & Goals
- Set financial targets (e.g., saving for a new branch).
- Track progress towards specific milestones.

### 4.7 AI Financial Advisor
- Integration with Gemini AI to analyze transaction patterns and provide strategic financial advice.

### 4.8 Reports & Accounting
- Generate monthly financial summaries.
- Approval workflow for ledger nodes.

## 5. UI/UX Design

### 5.1 Aesthetic
- **Theme**: Default Dark Mode with a "Cyber-Professional" look.
- **Typography**: Inter (Sans-serif) for high legibility.
- **Layout**: Sidebar navigation with a central "Command Center" workspace.
- **Components**: Large rounded corners (3xl/4xl), subtle glassmorphism, and bold high-contrast typography.

### 5.2 Navigation
- **Sidebar**: Primary navigation between modules (Dashboard, Income, Expense, Wallet, etc.).
- **Joystick Menu**: A floating action button (FAB) for rapid entry of sales, expenses, and AI insights.

---

## 6. Component Breakdown

- **Layout**: Wraps the application with navigation and header.
- **Dashboard**: The primary landing view with financial summaries.
- **TransactionModal**: Unified form for adding/editing income and expenses.
- **EqubTracker**: Specialized view for ROSCA management.
- **ChatCenter**: Internal communication hub for partners.
- **SecurityOverlay**: The PIN-entry lock screen.
- **CameraScanner**: (Placeholder/Future) For scanning receipts or QR codes.

---

## 7. Non-Functional Requirements

- **Performance**: Sub-second transitions between modules using React state.
- **Responsiveness**: Fully responsive layout adapting from mobile phones to ultra-wide monitors.
- **Offline Capability**: PWA manifest and service worker support for basic offline access.
- **Maintainability**: Strict TypeScript typing for all data structures and component props.
